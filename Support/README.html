SVK Bundle

This README just documents what the commands of this Bundle do and how you can fine-tune their behaviour; if you need general SVK Help/Introduction, you should look at the SVK homepage.

Commands

Add to Repository
selected project files/folders or the active file if it doesn't belong to a project
Adds the targets to the repository.
Blame
active file
Displays a line-by-line history of the file, showing you who wrote which line in what revision. Click a line to jump to it in an editor window. Hover over a revision number or author name to see when the corresponding line was last changed. The date format is adjustable. "wc" means the last change was in your working copy.
Commit
selected project files/folders or the active file if it doesn't belong to a project
Commits your changed files to the repository. A dialog asks you for the description of your changes; you may also choose to exclude files from the commit by un-checking them. If no files are selected or active, this command does nothing. If the target files have no local changes, nothing happens.
Diff With Working Copy (BASE)
active file
Displays the differences between the active file and an unaltered, pristine copy of the file at the same revision. Equivalent to svk diff -rBASE.
Info
selected project files/folders or the active file if it doesn't belong to a project
Displays detailed information about the selected files. Some parameters are configurable via shell variables; see the Configuration Options section below.
Log
selected project files/folders or the active file if it doesn't belong to a project
Displays the commit message history for the selected files. Some parameters are configurable via shell variables; see the Configuration Options section below. (TODO: if multiple files are selected, only the logs from the first are shown.)
Revert
active file
Revert changes made in checkout copies.
Status
directory of active file
Displays a list of files with changes in your working copy, along with the type of change for each file.
Update to Newest
selected project files/folders or the active file if it doesn't belong to a project
Updates the targets with the newest changes from the repository.

Configuration Options

These shell variables allow you to tweak the behaviour of the certain commands if need be. The default values should make sense for normal use.

$TM_SVK
svk
the path to your svk executable.
$TM_RUBY
ruby
Here you can tweak with what Ruby the formatting scripts will be executed.
$TM_SVK_DATE_FORMAT
not set (No Changes, just show what SVK shows)
A date format string that allows you to tweak the format in which Blame, Log and Info show you dates. See strftime(3) for what you can enter here.
$TM_SVK_CLOSE
false
With this option you can adjust whether the windows of Blame and Info close if you click on a link which opens a file in TM. Set it to 1 or true if windows should close or to something else if they should not.
$TM_SVK_LOG_RANGE
BASE:1
The default range to query for log messages.
$TM_SVK_LOG_LIMIT
9
The number of messages to show. 0 means no limit.
$TM_SVK_INFO_HIDE
not set (nothing)
Here you can adjust what keys of the info stream you don't want to see, it is case insensitive and a comma (,) separated list. If you enter a *, it is assumed that you want to hide all vars, if so, you can make some values visible again with $TM_SVK_INFO_SHOW.
$TM_SVK_INFO_SHOW
not set (everything else)
This is also a comma-separated list of keys, it just makes sense if you did hide all with the above var. * has no special meaning here. Everything you enter here will be shown if you want to hide everything else.
